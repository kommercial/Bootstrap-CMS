 91: ## System Requirements
 92: 
 93: Bootstrap CMS was designed to run on a Linux machine with PHP 5.4 and MySQL 5.5.  
 94: 
 95: * PHP 5.4.7+ or PHP 5.5+ is required.
 96: * MySQL 5.1+, 5.5+, or 5.6+ is required. MySQL 5.7+ may work, but is untested.  
 97: * You will need [Composer](https://getcomposer.org) installed to load the dependencies of Bootstrap CMS.  
 98: * You will need to configure the site in the app/config folder before production.  
 99: * Some features require a cache server like [Redis](http://redis.io) and a queuing server like [Beanstalkd](http://kr.github.io/beanstalkd).  
100: 
101: <br>
102: 
103: ## Installation
104: 
105: Please check the system requirements before installing Bootstrap CMS.  
106: 
107: 1. You may install by cloning from github, or via composer.  
108:   * Github: `git clone git@github.com:GrahamCampbell/Bootstrap-CMS.git`
109:   * Composer: `composer create-project graham-campbell/bootstrap-cms --prefer-dist`
110: 2. From a command line open in the folder, run `composer install`.  
111: 3. Enter your database details into `app/config/databse.php`.  
112: 4. You can run `php artisan app:install` to setup and seed your database.  
113:   * Make sure you setup your database config in database.php. 
114: 5. Finally, setup an [Apache VirtualHost](http://httpd.apache.org/docs/current/vhosts/examples.html) to point to the "public" folder.
115:   * For development, you can simply run `php artisan serve`
116: 6. Additionally, you may to setup queuing and/or caching (see below).  
117:

120: ## Getting Queuing Working
121: 
122: Note that `beanstalkd` requires a local server, while `sqs` and `iron` are cloud based.  
123: 
124: 1. Choose your poison - I\'d recommend [IronMQ](http://www.iron.io/mq).  
125: 2. Enter your queuing server details into `app/config/queue.php`.  
126: 3. You can also set a separate mail queue in `app/config/mail.php`.  
127: 
128: <br>
129: 
130: ## Getting Caching Working
131: 
132: Note that caching will not work with Laravel\'s `file` or `database` cache drivers.  
133: 
134: 1. Choose your poison - I\'d recommend [Redis](http://redis.io).  
135: 2. Enter your cache server details into `app/config/cache.php`.  
136: 3. Enable Bootstrap CMS\'s caching in `app/config/cms.php`.

140: ## Getting Google Analytics Working
141: 
142: Bootstrap CMS natively supports Google Analytics.  
143: 
144: 1. Setup a web property on Google Anaytics.  
145: 2. Enter your tracking id into `app/config/analytics.php`.  
146: 3. Enable Google Analytics in `app/config/analytics.php`.

150: ## Getting Themes Working
151: 
152: Bootstrap CMS also ships with 13 themes from Bootswatch.  
153: 
154: 1. You can set your theme in `app/config/theme.php`.  
155: 2. You can also set your nav bar style in `app/config/theme.php`.  
156: 3. After making theme changes, you will have to run `php artisan app:update`.
